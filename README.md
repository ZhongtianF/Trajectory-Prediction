# 1. Introduction

Human trajectory prediction consists in forecasting the future path of pedestrians given their past observed positions.

This problem is important for many applications such as:

- autonomous driving
- robotics navigation
- surveillance systems
- crowd analysis

In this project, we predict the future trajectory of pedestrians using the **ETH/UCY datasets**.

We implemented and compared several models:

1. Constant Velocity (baseline)
2. LSTM Encoder–Decoder
3. Scene-aware LSTM (using visual context)
4. Conditional GAN for multimodal prediction

---

# 2. Dataset

We use the **ETH / UCY pedestrian trajectory datasets**.

Scenes used in this project:

- ETH
- HOTEL
- UNIVERSITY
- ZARA01
- ZARA02

Each trajectory sample contains:

Observed trajectory:
8 timesteps

Future trajectory:
12 timesteps

Coordinates:

(x, y) positions of pedestrians

Dataset files are stored as SQLite databases.

Example table:
dataset_T_length_20delta_coordinates


Columns:

- pos_x
- pos_y
- pos_x_delta
- pos_y_delta
- ped_id
- frame_num
- data_id

# 5. Baseline Model

Constant Velocity assumption.

The model predicts the future trajectory using the last observed velocity.
python src/baseline_cv.py

Evaluation metrics:

- ADE (Average Displacement Error)
- FDE (Final Displacement Error)

---

# 6. LSTM Encoder–Decoder

The trajectory is encoded using an LSTM encoder.

The future trajectory is generated by a decoder LSTM.

Training:
python -m src.train_lstm --scene eth

Evaluation:
python -m src.eval_lstm --scene eth

---

# 7. Scene-aware LSTM

To integrate scene context, we extract visual features using a **ResNet18 encoder**.

Scene image → ResNet18 → 512-dimensional feature

The scene feature is fused with the trajectory encoder state.

Training:
python -m src.train_scene_lstm --scene eth

---

# 8. GAN Trajectory Model

To model multiple possible futures, we implement a **conditional GAN**.

Generator input:

- observed trajectory
- scene feature
- random noise

Output:

future trajectory

Discriminator input:

- full trajectory (observed + predicted)
- scene feature

Training:
python -m src.train_gan --scene eth

Evaluation (Best-of-K):
python -m src.eval_gan --scene eth --K 20

Metrics:

ADE@K  
FDE@K  
Diversity

# 11. Metrics

We use standard trajectory prediction metrics.

ADE (Average Displacement Error)
ADE = mean distance between predicted and ground truth trajectory

FDE (Final Displacement Error)
distance between predicted final position and ground truth final position

For GAN models we also compute:

Best-of-K ADE / FDE
minimum error over K generated trajectories

Diversity
average pairwise distance between generated trajectories
